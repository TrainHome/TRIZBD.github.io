4. Схема связей и тип связей

Клиент (1) ──────< (М) Заказ
Сотрудник (1) ──< (М) Заказ
Категория (1) ──< (М) Товар
Заказ (М) >─────< (М) Товар (реализовано через СоставЗаказа)

📐 Типы связей:

  • Клиент – Заказ: один-ко-многим (один клиент → много заказов)
  • Сотрудник – Заказ: один-ко-многим (один сотрудник → много заказов)
  • Категория – Товар: один-ко-многим (одна категория → много товаров)
  • Заказ – Товар: многие-ко-многим

🔧 Реализация связи «Заказ – Товар»:

Через промежуточную таблицу СоставЗаказа, где хранятся:
• Количество товара в заказе
• Цена на момент покупки (фиксируется, т.к. текущая_цена в Товаре может меняться)

6. ER-модель (текстовое описание)

[Клиент]─────────<[Заказ]>──────────[Сотрудник]
              |
              | (через СоставЗаказа)
              V
            [Товар]──────────>[Категория]

📖 Пояснение к ER-модели:

  • У каждого Заказа есть один Клиент и один Сотрудник (принявший заказ).
  • У каждого Товара есть одна Категория.
  • Между Заказом и Товаром – связь многие-ко-многим через таблицу СоставЗаказа, где дополнительно хранится Количество и Цена_на_момент_покупки (выполнено требование п. 7).
